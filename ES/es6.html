    # 5. 箭头函数

    1. this 是静态的，始终指向函数声明时所在作用域下的 this 的值。
    2. 不能作为构造函数实例化对象
    3. 不能使用 arguments 对象

    # 注意：

    箭头函数没有自己的 this，它的 this 是继承来的，默认指向它定义的时候所处的对象。

    # 适用场景：

    与 this 无关的回调。 定时器、数组的方法回调等；

    # 不适用场景：

    与 this 有关的回调。 事件回调、对象的方法等。

    # 9. Symbol (第7种数据类型)

    1. Symbol的值是唯一的，可以用来解决命名冲突问题
    2. Symbol值不能与其他数据类型进行运算
    3. 内部属性不能通过使用for...in来遍历，但是可以使用Reflect.ownKeys来获取对象的所有键名
    let s = Symbol();
let m = Symbol.for("key");

    # 10. 迭代器 Iterator

    1. ES6 创造了一种新的遍历命令 for...of 循环，Iterator 接口主要供 for...of 消费
    2. 原生具备Iterator接口的数据： Array, Arguments, Set, Map, String, TypedArray, NodeList

    # 工作原理：

    1. 创建一个指针对象，指向当前数据结构的起始位置
    2. 第一次调用对象的next方法，指针自动指向数据结构的第一个成员
    3. 不断调用next方法，依次往后遍历，返回{value:"", done:false}对象
    4. 遍历完成，返回{value: undefined, done: true}对象
    let foo = {
	name: "foo",
	stus: ["aaa", "bbb", "ccc", "ddd"],
	[Symbol.iterator]: function () {
	    let idx = 0;
	    let len = this.stus.length
	    return {
	        next: () => {
	            if (idx < len) {
	                return {value: this.stus[idx++], done: false};
	            } else {
	                return {value: undefined, done: true};
	            }
	        }
	    }
	}
}

    # 12. Promise

    封装ajax：

    const p = new Promise((resolve, reject)=>{
	const url = "";
	const xhr = new XMLHttpRequest();
	xhr.open(url);
	xhr.send();
	xhr.onreadystatechange = ()=>{
		if(xhr.readyState === 4) {
			if(xhr.status >= 200 && xhr.status < 300) {
				resolve(xhr.response);
			} else {
				reject(xhr.status);
			}
		}
	}
});

    # 18. 模块化

    模块化是指将一个大的程序文件，拆分成许多小的文件，然后将小文件组合起来。

    好处：

    1. 防止命名冲突
    2. 代码复用
    3. 高维护性

    ES6之前没有模块化，由社区维护，规划产品：

    • CommonJS => NodeJS Browserify
    • AMD => requireJS
    • CMD => seaJS
